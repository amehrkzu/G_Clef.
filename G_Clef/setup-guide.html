公開・運用ガイド

GitHubの登録から、サイトの公開・日々の更新までを順番に説明します。

このガイドで使うファイルについて

お渡しするフォルダの中身は以下の構成になっています。すべてまとめて使いますので、フォルダの中身は動かさずそのままアップロードしてください。

G_Clef/
  ├ index.html　　… トップページ
  ├ order.html　　… 注文票作成ページ
  ├ images/　　… サイトで使う画像一式
  └ js/　　　… サイトの動きを作るプログラム一式

STEP 1GitHubアカウントを作る

GitHub（ギットハブ）は、ホームページのデータを保存しておき、そのまま無料でインターネット上に公開できるサービスです。まずは無料のアカウントを作ります。

  1. https://github.com/ にアクセスします。
  2. 右上（またはトップページ中央）の「Sign up」ボタンを押します。
  3. メールアドレス・パスワード・ユーザー名を入力し、画面の指示に沿って進めます（メール認証コードの入力が必要です）。
  4. 登録が完了すると、GitHubの管理画面（ダッシュボード）が表示されます。
ここで決めたユーザー名は、後で公開するサイトのURLの一部になります。忘れないようにメモしておいてください。

STEP 2新しいリポジトリを作る

リポジトリとは、ファイルを保存しておく「箱」のようなものです。ホームページ1つにつき、リポジトリを1つ用意します。

  1. GitHubの画面右上の「+」アイコン →「New repository」を選びます。
  2. Repository name（リポジトリ名）を入力します。
  3. Public（公開）が選ばれていることを確認します。
  4. 他の項目はそのままで、一番下の「Create repository」ボタンを押します。
リポジトリ名を ユーザー名.github.io（例：ユーザー名が piiarulaq なら piiarulaq.github.io）にすると、公開URLが一番シンプルな形になります。それ以外の名前でも問題なく公開できます。

STEP 3ファイルをアップロードする

  1. 作成したリポジトリの画面で「Add file」→「Upload files」を選びます。
  2. お渡ししたG_Clefフォルダの中身（index.html・order.html・imagesフォルダ・jsフォルダ）を、そのままドラッグ＆ドロップでアップロード画面に入れます。
  3. フォルダごとドラッグすれば、中の構成を保ったままアップロードされます。
  4. 下の方にある「Commit changes」ボタンを押して確定します。
G_Clefフォルダそのものではなく、フォルダの中身をアップロードしてください。フォルダごと1つ入れてしまうと、サイトが正しく表示されません。

STEP 4GitHub Pagesを有効にする

アップロードしたファイルを、実際にインターネット上のホームページとして公開する設定です。

  1. リポジトリ画面上部の「Settings」タブを開きます。
  2. 左側メニューの「Pages」を選びます。
  3. 「Build and deployment」の「Branch」で main を選び、フォルダは / (root) のまま、「Save」を押します。
  4. 1〜2分待つと、画面上部に公開されたURL（例：https://ユーザー名.github.io/リポジトリ名/）が表示されます。

STEP 5サイトを確認する

  1. STEP 4で表示されたURLをブラウザで開きます。
  2. トップページが表示され、ナビゲーションの「注文票作成」からもう一方のページに移動できることを確認します。
更新した内容がすぐに反映されない場合は、数分待ってからブラウザの再読み込み（Ctrl + Shift + R）を試してください。GitHub Pagesの反映には少し時間がかかることがあります。

日々の運用について

在庫一覧・お知らせの内容は、HTMLファイルではなくGoogleスプレッドシートで管理しています。担当者から共有されたスプレッドシートを開き、該当のシート（タブ）に文字を入力・編集するだけで、サイト側に自動的に反映されます。ファイルのアップロードやGitHubの操作は不要です。

  • 在庫の追加・変更・削除 → 各カテゴリのシートを編集
  • お知らせの追加・変更 → 「お知らせ」シートのタイトル列・本文列を編集
スプレッドシートの編集権限がない場合は、担当者に共有をお願いしてください。

困ったときは

  • サイトが表示されない → STEP 3でアップロードした場所（フォルダの中身が正しいか）とSTEP 4の設定を再確認してください。
  • 画像が表示されない → imagesフォルダが他のファイルと同じ階層にアップロードされているか確認してください。
  • お知らせ・在庫が反映されない → スプレッドシートの該当セルが空欄になっていないか、数分待ってから再読み込みしてみてください。
